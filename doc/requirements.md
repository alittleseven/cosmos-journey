# 需求文档（Requirements）

> 项目：Cosmos Journey · 旅游打卡 Web 应用
> 版本：v0.2 · 状态：已上线（GitHub Pages）

## 1. 背景与目标

记录旅游景点与特殊位置的轻量 Web 应用。用户在地图上一键打卡，附带照片、笔记、
评分与分类，并把去过的地方汇成个人旅行足迹。

**核心目标**
- 低门槛:打开网页即用,无需注册登录
- 快速记录:从"想记"到"记完"尽量少步骤
- 可视化:地图 + 足迹连线 + 统计,直观回看旅程

## 2. 用户角色

| 角色 | 说明 |
|------|------|
| 访客/记录者 | 唯一角色。数据保存在本人浏览器本地,自管自看 |

> 当前为单机本地应用,暂无多用户/协作概念(见 Roadmap 云端同步)。

## 4. 非功能需求（NFR）

- **性能**:首屏 JS gzip ≈ 78KB;地图标记使用轻量 divIcon
- **兼容性**:现代浏览器(Chrome/Edge/Safari/Firefox 近两年版本)
- **可用性**:无后端依赖,GitHub Pages 静态托管,高可用
- **数据安全**:数据仅存于用户浏览器 `localStorage`,不上传服务器
- **可维护性**:TypeScript 严格模式 + 组件化 + Pinia 单一数据源
- **可移植性**:存储层经 Pinia store 封装,未来可平滑替换为云端

## 5. 关键用户故事

- 作为记录者,我想**在地图上点一下就把当前位置存为打卡点**,这样记录很快。
- 作为记录者,我想**给每个点加照片和评分**,方便日后回忆质量高低。
- 作为记录者,我想**按分类筛选和搜索**,快速找到想看的地点。
- 作为记录者,我想**看到去过的地方连成一条线**,直观感受旅程轨迹。
- 作为记录者,我想**看到统计**(去过多少、什么类型最多),获得成就感。

## 6. 范围

**包含**:单机本地的地图打卡、照片/笔记/评分、足迹、统计、静态部署。

**不包含(当前版本)**:账号体系、服务端存储、多端同步、社交分享、离线 PWA。

## 7. 约束与假设

- 用户允许浏览器定位(否则用手动点选地图)。
- `localStorage` 容量有限(约 5MB),照片已压缩;海量数据需迁云端。
- 高德瓦片为 GCJ-02 坐标系,与 OSM(WGS-84)存在偏移(详见架构/技术文档)。

## 8. 验收标准（示例）

- 新增打卡后,刷新页面数据仍在,且地图与列表同步显示。
- 切换分类筛选时,地图标记与列表条目一致变化。
- 开启"足迹"后,≥2 个点按到访时间连成虚线。
- push 到 `main` 后,GitHub Actions 自动部署并更新线上站点。
